Утилита - jconsole.

Было проведено два запуска приложения с SerialGC и ParallelGC с добавлением и удалением элементов.

SerialGC:

При добавлении элементов:
Пробовал добавить 1млн элементов, все нормально, cpu поднялось до 0.3% всего лишь, heap memory до 37мб.
Пробовал добавить 1млрд элементов, cpu поднялось до 3%, а heap memory до 310МБ, после чего выбросило исключение ошибки OutOfMemory.


При удалении элементов:
При просмотре всех элементов(а их было 1млн), cpu скакнул до всего лишь 4.6%, a heap до 28МБ.
При удалении 1млн элементов, cpu до 13%, heap до 23МБ всего лишь, при этом удаление заняло полторы минуты где-то.
-----------------------------------
ParallelGC:

При добавлении элементов:
Пробовал добавить 1млн элементов, все нормально, cpu поднялось до 0.5% всего лишь, heap memory до 50мб.
Пробовал добавить 1млрд элементов, cpu подскочило до 40%, а heap memory до 1.2гб, после чего выбросило исключение ошибки OutOfMemory.


При удалении элементов:
При просмотре всех элементов(а их было 1млн), cpu скакнул до всего лишь 6%, a heap до 250МБ.
При удалении 1млн элементов, cpu скакнуло так же до 13%, heap до 284МБ, при этом удаление заняло 4-5 минут.



Итого, для трекера в текущем виде, лучше всего использовать простой SerailGC, т.к. для одних и тех же значений добавления и удаления, отработало и быстрее, и экономнее чем в случае ParallelGC.
